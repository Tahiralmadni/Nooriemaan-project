📄 Project Ka Vision Aur Maqsad

Nooriemaan Digital Portal ek azeem-o-shaan ERP (Enterprise Resource Planning) system hai jo ke educational institutes ke har department ko digitalize karne ke liye banaya ja raha hai. Is project ka vision sirf data entry nahi, balke automated decision-making aur minute-level tracking hai.

Hamara maqsad ye hai ke idaray ka har ek fard, chahe woh ustad ho, khadim ho, ya student, unka pura record system mein digital maujood ho. Is document mein hum ne 50+ pages ka detail blueprint banaya hai jo ke is software ka hissa honge.

Software ki base dunya bhar ke modern software design patterns par rakhi gayi hai takay ye fast, secure, aur future-proof rahe. Ye software sirf ek website nahi hai balkay aik pura digital brain hai.

📄 Tech Stack: React JS Engine

Frontend Core: React 19.2.0

Is software ka pura frontend React JS par banaya gaya hai. React dunya ka sab se popular aur fast framework hai jo ke Facebook ne develop kiya hai. Iska Virtual DOM mechanism software ko intehai fast banata hai.

React istemal karne ki wajah ye hai ke jab user kisi button par click karta hai, tou poora page reload nahi hota, balke sirf woh makhsoos hissa update hota hai. Is se user ka waqt bachta hai aur experience smooth rehta hai.

Hum ne isme "Concurrent Rendering" use ki hai jo ke heavy calculations ko background mein handle karti hai bina UI ko freeze kiye.

📄 Tech Stack: Firebase Cloud

Backend & Database: Google Firebase

Data ki hifazat aur speed ke liye hum ne Google ka Firebase Cloud istemal kiya hai. Firebase aik serverless architecture hai jo ke hamare data ko cloud par securely store karta hai.

Firestore Database: Ye NoSQL database hai jo ke real-time synchronization provide karta hai. Agar admin ek mobile se hazri lagata hai, tou foran dashboard par nazar ajati hai bina refresh kiye.

Authentication: Users ka login system Google ki security layers par mabni hai, jo ke passwords ko encrypt karke rakhta hai.

📄 System Architecture & Folder Map

Software ko maintainable rakhne ke liye hum ne professional structure follow kiya hai:

  • /src/pages: Yahan 22+ pages ka code maujood hai jo ke business logic handle karte hain.
  • /src/components: Yahan re-usable UI elements (Buttons, Inputs, Modals) hain jo poore software mein aik jaisa design maintain rakhte hain.
  • /src/hooks: Custom logic hooks (jaise useStaffData) jo database se data fetch aur calculate karte hain.
  • /src/utils: Reporting aur time parsing ke liye helper functions.

Is structure ki wajah se software mein bug dhundna aur naye features add karna bohot asan hai.

📄 Security Protocols: Encryption

Data security hamari top priority hai. Hum ne system mein "Multi-Layered Security" implement ki hai:

  • JWT Token System: Har user ko login ke waqt aik unique encrypted token milta hai jo ke browser ki memory mein securely store hota hai.
  • HTTPS/SSL: Tamam data transfer encrypted tunnel ke zariye hota hai takay koi beech mein data chura na sake.
  • Sanitized Inputs: Har form input ko database mein save karne se pehle sanitize kiya jata hai takay SQL Injection jaise attacks se bacha ja sake.
Nooriemaan Digital Portal — Hanzalah Tahir

📄 Security Protocols: Database Rules

Firestore Database ke liye hum ne "Strict Security Rules" likhe hain:

  • Role-Based Access (RBAC): Sirf wohi log data dekh sakte hain jin ke pas permission hai. User-A kisi dusre admin ka data edit nahi kar sakta.
  • Validator Rules: Agar koi galat data (jaise negative salary) bhejne ki koshish kare, tou database us request ko reject kar deta hai server-side par.
  • Audit Readiness: System har action ka log rakhta hai takay future mein audit ho sake.

📄 Module 5: Auto-Late Detection Algorithm

Hamara system "Time-Math" use karta hai takay late minutes calculate ho saken:
